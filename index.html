Is it actually?

gatekeep is a deliverable-contract checker for long-horizon agent tasks. Declare what a task must produce in one YAML file. Run one command. Get a pass/fail report — no LLM call, no API key, no network — in milliseconds, that catches deliverable drift before a human (or a costly downstream test run) ever has to.

Try it right now

Two install paths, both verified end-to-end to take under five minutes from a cold machine.

# Path 1 — zero install, zero dependencies (curl one file, run it)
curl -fsSL https://raw.githubusercontent.com/REPLACE_ME/gatekeep/main/gatekeep_single.py -o gatekeep_single.py
python3 gatekeep_single.py init
python3 gatekeep_single.py check
# → prints a markdown report, exits 1 if any required deliverable fails
# Path 2 — pip install, full CLI on PATH
git clone https://github.com/REPLACE_ME/gatekeep.git
cd gatekeep/gatekeep && pip install .
gatekeep init && gatekeep check

Both paths read the exact same gatekeep.yml contract format and are kept in lockstep by an automated parity test suite — no "the quick path is the lesser path" asterisk.

What a contract looks like

No prompts, no rubric scoring by an LLM. Pure, boring, reproducible checks.

name: my-project-deliverables
deliverables:
  - id: api-patch
    description: "the agent's code change is real, not empty or test-only"
    severity: required
    checks:
      - kind: valid_unified_diff
        path: "patch.diff"
        forbid_test_only: true
  - id: readme
    severity: required
    checks:
      - kind: min_words
        path: "README.md"
        min_words: 50
      - kind: no_placeholder_text
        path: "README.md"

Measured, not asserted

We ran gatekeep's deterministic engine against 300 real, publicly published agent submissions to SWE-bench Lite — actual patches and trajectories produced by SWE-agent + Claude 3.5 Sonnet, pulled from the official, public swe-bench/experiments repository, scored against ground truth from princeton-nlp's own SWE-bench evaluation harness. Why this, not an LLM judge

Reproducible forever

Same contract, same filesystem state, same verdict — for anyone, on any day, with or without an API key.

Zero added cost

No model call in the loop. Checks run in milliseconds and can gate CI without adding latency or spend.

Auditable rules

Every check is a small, readable function. You can read the entire decision logic in one sitting.

Composable with judges

Use gatekeep for the hard, structural guarantees; add an LLM-judge check only for genuinely soft rubric criteria, on your own terms.

CI-native

One file, one step, fails the build — drop-in GitHub Action included.

No telemetry

Fully offline by default. A governance tool that doesn't phone home.
