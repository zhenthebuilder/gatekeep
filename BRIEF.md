# Long-Horizon Research Brief

## High-level goal

Build something that **people can use to better control and govern the
deliverables of long-horizon agent tasks**. The artifact should be adoptable
in real workflows — not just a paper.

You may reframe the problem if a different framing serves the goal better.
The high-level goal is the only fixed thing.

## What you must deliver

By the end of the run, the workspace must contain all five of the following.
Missing any one is a failed run.

1. **A deployable form of your system** — distributed in whatever shape you
   choose. A stranger on the internet must be able to install or adopt it
   in under 5 minutes from a single command, link, or download. "It compiles
   on my machine" is not deployable; "anyone can use it today" is.

2. **A research paper** in NeurIPS 2026 LaTeX format (or another recognized
   conference format). Must contain a Results section grounded in real
   numbers from your own benchmark run. No fabricated numbers.

3. **An A/B run on a public, recognized benchmark**, with reproducible
   scoring code and raw logs in the workspace. If no public benchmark fits,
   you must still report at least one public-benchmark result to anchor
   your claim.

4. **A marketing plan** as markdown.

5. **A landing page** as `index.html`, openable directly in a browser
   without a build step. Must include a real install / try-it path that
   points to deliverable #1.

## Rules of the run

You may read public docs, papers, repos, datasets, and benchmarks. You may
run experiments in isolated environments. You may use sub-agents, MCP
servers, and external APIs.

You may not:

- Talk to the user during the run. Make documented decisions and proceed.
- Fabricate numbers, tool output, or citations.
- Modify this BRIEF.md.
- Read or reuse reference solutions for this problem that may exist
  elsewhere on this machine. If you find any, do not read them.

## Budget

8 hours of wall time.
